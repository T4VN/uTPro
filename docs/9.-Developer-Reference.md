# 9. Developer Reference — Helpers & Extensions

uTPro ships a set of Razor/C# helpers used throughout the views. This page is a quick reference for developers building components and templates. Add `@using uTPro.Extension;` (and `@using uTPro.Extension.CurrentSite;`) where needed — most are already imported via `_ViewImports.cshtml`.

> The **Script Queue** helpers (`QueueScript` / `QueueStandaloneScript`) have their own page — see [5. Script Queue](5.-Script-Queue).

## 9.1 Current site & item context

`ICurrentSiteExtension` (injected as `CurrentSite` in views) gives per-request site context:

| Member | Returns / does |
|--------|----------------|
| `CurrentSite.GetItem()` | The current site's key nodes (see below) |
| `CurrentCulture` | The active `CultureInfo` |
| `DefaultCulture` | The site's default culture |
| `GetCultures()` | All published cultures (used for the language switcher) |
| `GetDomains(bool isGetAll)` | Configured domains for the site |
| `GetUrlWithCulture(content, culture, mode)` | URL of a node in a specific culture |
| `GetDictionaryValue(key, valueDefault = "", showKey = false)` | Dictionary translation lookup |
| `SetCurrentCulture(culture)` | Override the request culture |

`CurrentSite.GetItem()` returns an `ICurrentItemExtension` exposing the site's structural nodes:

| Member | Node |
|--------|------|
| `Root` | `GlobalRoot` (also `Root.Name` — used as the per-site CSS/JS folder name) |
| `FolderSite` | `GlobalFolderSites` |
| `FolderSettings` | `GlobalFolderSettings` (favicon, responsive-image & SMTP settings live here) |
| `PageHome` | The site's Home page |
| `Current` | The current page |
| `PageErrors` | The error page node |

```razor
@inject ICurrentSiteExtension CurrentSite
<link rel="stylesheet" href="~/css/@CurrentSite.GetItem().Root.Name/Components/MyComponent.css" />
<span>@CurrentSite.GetDictionaryValue("uTPro.Form.ContactUs.Message")</span>
```

## 9.2 Content inheritance

`ContentExtensions` resolves values inherited down the content tree (self → ancestors), with per-request caching:

```razor
@* First ancestor (or self) that has "topComponent" *@
var top = Model.ValueInherited<BlockGridModel>("topComponent");

@* The node that holds the value, for a set of aliases (first match wins) *@
var (alias, node) = Model.Inherited("bannerImage", "defaultImage");
```

Used by the page templates for `topComponent` / `bottomComponent` so a value set on `Home` cascades to child pages.

## 9.3 Media & responsive images

**`ResponsiveImg(...)`** (`ResponsiveImageExtension`) renders a responsive `<img>` (WebP + `srcset`), driven by [Responsive Image Settings](8.-Global-Settings#83-responsive-image-settings):

```razor
@image.ResponsiveImg(
    alt: "Hero banner",
    cssClass: "hero-img",
    sizes: "100vw",
    priority: true)      @* LCP/above-the-fold: eager + fetchpriority=high *@
```

Key parameters: `alt` (falls back to the media's `altText`), `cssClass`, `sizes` (default `100vw`), `lazy` (default `true`), `priority` (LCP images), `includeDimensions` (default `true`, prevents layout shift), `quality` (default `80`). Non-image media or the toggle being off emits the original URL untouched.

**`ValueToMedia*`** (`ValueToMediaUrlExtension`) returns a media **URL string** for a property alias, resized to a breakpoint and converted to WebP (images only; other media returns the absolute URL):

| Method | Target width |
|--------|--------------|
| `value.ValueToMediaDesktop("alias")` | 2048 |
| `value.ValueToMediaTablet("alias")` | 1500 |
| `value.ValueToMediaMobile("alias")` | 800 |
| `value.ValueToMedia("alias", format: "WebP")` | original (format-converted) |

```razor
<img src="@Model.ValueToMediaDesktop("image")" alt="@Model.Value("title")" />
```

## 9.4 HTML & string helpers

| Helper | Signature | Purpose |
|--------|-----------|---------|
| `ToHtml()` | `string?.ToHtml()` / `IHtmlEncodedString?.ToHtml()` | Renders CMS HTML with a **best-effort** neutraliser (strips `script`/`style`/`iframe`/`object`/`embed`/`base`/`form`, inline `on*` handlers and `javascript:`/`vbscript:`/`data:` URIs). **Not** a full sanitizer — don't rely on it for untrusted input. |
| `ToInline()` | `string?.ToInline()` / `IHtmlEncodedString?.ToInline()` | Collapses newlines to spaces (single-line output). |
| `LinkToString()` | `Link?.LinkToString()` | Safe URL string from an Umbraco `Link` (handles null/empty). |
| `ToGuid()` | `Udi?.ToGuid()` | `GuidUdi` → `Guid` (`Guid.Empty` when null). |
| `ToJsonString()` | `IDictionary<string,string>.ToJsonString()` | Compact JSON serialisation. |

## 9.5 Sections (CSS placement)

`HtmlSectionBlockExtensions.SetSection(key, position)` lets a component register CSS/markup that the layout renders in the right place (deduplicated by `key`):

```razor
@using (Html.SetSection(Model.GetType().ToString(), HtmlSectionBlockExtensions.Position.BodyBottom))
{
    <link rel="stylesheet" href="~/css/@CurrentSite.GetItem().Root.Name/Components/MyComponent.css" />
}
```

For **JavaScript**, use the Script Queue instead of `<script>` tags — see [5. Script Queue](5.-Script-Queue).

## 9.6 Colour & file utilities

- **`ColorHelper`** — `HexToRgb(hex)` (`#RGB`/`#RRGGBB` → `Color`), `RgbToHex(r,g,b)` / `RgbToHex(Color)`. Handy for CMS colour-picker values (e.g. the section wrapper in [7.4](7.-Content-Editing#74-building-a-page-with-the-block-grid)).
- **`FileNameHelper.GetValidFileName(name)`** — sanitises a file name.
- **`MimeTypeMap`** — `GetMimeType(path)`, `TryGetMimeType(path, out mime)`, `GetExtension(mime)`.

## 9.7 Block preview aware components

Components can render a lighter output in the backoffice preview (CSS only, no JS):

```razor
@if (Context.Request.IsBlockPreviewRequest())
{
    <link rel="stylesheet" href="~/css/@CurrentSite.GetItem().Root.Name/Components/MyComponent.css" />
}
else
{
    // full render + Html.QueueScript(...) / QueueStandaloneScript(...)
}
```

See [10.3 Block Preview](10.-Backoffice-Tools#103-block-preview).
