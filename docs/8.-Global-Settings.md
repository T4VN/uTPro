# 8. Global Settings (CMS-driven)

Several site behaviours are controlled by dedicated **backoffice settings nodes** rather than `appsettings.json`, so editors can change them without a redeploy. They live under **Folder Settings** (site-wide) or as compositions on the **Home** / **Folder Settings** node.

> Security headers are also CMS-driven; they have their own section — see [4.4.1](4.-Configurations#441-security-headers-backoffice-driven).

## 8.1 Site Settings — favicon, logo, title suffix

Node: **Site Settings** (`globalSiteSettings`), under Folder Settings.

| Field | Alias | Purpose |
|-------|-------|---------|
| Favicon.ico | `siteSettingsfaviconIco` | Site favicon. Falls back to `/favicon-default.ico` when empty. |
| Logo | `siteSettingsLogo` | Site logo media. |
| Title Suffix | `siteSettingsTitleSuffix` | Appended to every page title as `Page Title | Suffix` (see [7.7](7.-Content-Editing#77-seo-per-page)). |

The favicon is resolved per request; if the node or the image is missing, the default icon is served.

## 8.2 Sitemap & Robots Settings

Node: **Sitemap & Robots Settings** (`globalSitemapRobotsSettings`), under Folder Settings.

| Field | Alias | Purpose |
|-------|-------|---------|
| Content | `robotstxtContent` | Raw `robots.txt` body. **When empty**, a default is generated: `User-agent: *` plus a `Sitemap:` line pointing at the current host's `/sitemap.xml`. |
| Ignored Document Types | `sitemapIgnoredDocumentTypes` | Doc-type aliases to exclude from `sitemap.xml`. |
| Max Sitemap Depth | `sitemapMaxSitemapDepth` | Limit how deep the sitemap crawls the tree. |

`robots.txt` and `sitemap.xml` are served at the site root and cached ~30 minutes. Per-page sitemap/indexing toggles are set on the page itself (see [7.7](7.-Content-Editing#77-seo-per-page)).

## 8.3 Responsive Image Settings

Composition on the **Folder Settings** node (`globalResponsiveImageSettings`). Controls uTPro's responsive-image helper.

| Field | Alias | Purpose |
|-------|-------|---------|
| Turn Off Responsive Image | `repImgTurnOffResponsiveImage` | Master toggle. **OFF = responsive images ON**: image URLs are resized (bounded by the longest edge) and served as WebP with a `srcset`. ON = original URL emitted untouched. |
| Extend Sizes SrcSet | `repImgExtendSizesSrcSet` | Extra widths for the `srcset` (e.g. `1200w`, `1024w`, `768w`). When empty, defaults are **1200 / 1024 / 768**. |

Images are never upscaled beyond their intrinsic size. Rendered from views via `ResponsiveImg(...)` — see [9.3](9.-Developer-Reference#93-media--responsive-images).

## 8.4 Contact Form & Cloudflare Turnstile

The **Contact Form** component (`uTPro__ContactForm`, [7.6](7.-Content-Editing#76-component-catalogue)) renders a form selected in its **Contact Form** field. Forms are built with the Simple Form Builder and can include a **Cloudflare Turnstile** anti-spam field.

**Turnstile key resolution** (per field, first match wins):

1. Value entered on the form field's **Site Key** / **Secret Key** settings.
2. Fallback to `appsettings.json`:

```json
{
  "uTProFormAddon": {
    "Turnstile": {
      "SiteKey": "0x4AAAAAAA...",
      "SecretKey": "0x4AAAAAAA..."
    }
  }
}
```

3. If both are empty, the widget is **not rendered**.

- **Site Key** is public (sent to the browser); **Secret Key** is server-side only (used to verify the token) and never emitted.
- Set global keys in `appsettings` so most forms work out of the box; a specific form can override them on its own field.

**SMTP** for form notifications is configured on the **SMTP Settings** composition (`globalSmtpSettings`): `smtpHost`, `smtpPort`, `smtpEnableSsl`, `smtpEmailFrom`, `smtpCredentialUsername`, `smtpCredentialPassword`.

## 8.5 Hidden container URLs

Organisational nodes should not expose a public URL. `HiddenUrlProvider` suppresses URLs for these built-in container doc types (always hidden):

`globalRoot`, `globalFolderSites`, `globalFolderNavigationLinkForSite`, `globalFolderSettings`.

To hide additional container doc types, list their aliases in `appsettings.json`:

```json
{
  "uTPro": {
    "HiddenUrls": {
      "DocumentTypeAliases": [ "myContainerDocType" ]
    }
  }
}
```

Matching nodes show a "This node is a container and has no URL." message in the backoffice **Links** panel instead of a navigable URL.

## 8.6 Error page

Node: **Page Error** (`globalPageError`), typically under Home. Provides the content for **404 / 500** responses with proper HTTP status codes (no soft-404s). It carries SEO meta and sitemap fields like a normal page, so the error page can be excluded from indexing. Wiring is handled in code (`ErrorController` / status-code re-execute); see [10.2](10.-Backoffice-Tools#102-error-pages) and [4.6](4.-Configurations#46-seo-configuration).
