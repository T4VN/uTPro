# 3.4 Program.cs — Clean Entry Point

The application entry point is intentionally minimal. All configuration logic is organized into extension methods in the `Startup/` folder:

```csharp
using uTPro.Project.Web.Configure;
using uTPro.Project.Web.Startup;

var builder = WebApplication.CreateBuilder(args);

builder.ConfigureTempPath();                              // Hosting
builder.ConfigureUmbraco();                               // Umbraco CMS
builder.Services.AddPerformanceServices(builder.Environment); // Performance
builder.Services.AddSecurityServices(builder);            // Security
builder.Services.AddRenderingDefaults();                  // MVC
builder.Services.AddControllers();
builder.Services.AddBlockPreviewServices();               // Block Preview

var app = builder.Build();
await app.BootUmbracoAsync();
app.ConfigurePipeline();                                  // Middleware
await app.RunAsync();
```

Each `Startup/*.cs` file is self-contained and can be modified independently.

# 3.5 Middleware Pipeline Order

Configured in `Startup/PipelineSetup.cs`:

1. **Security Headers** — `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Permissions-Policy`, `HSTS`
2. **Browser Cache-Control** — `stale-while-revalidate` for HTML pages (via `OnStarting` callback)
3. **HTTPS Redirection** + **HSTS**
4. **Status Code Pages** — `UseStatusCodePagesWithReExecute("/error/{0}")` (preserves 404/500 status)
5. **WebOptimizer** — CSS/JS minification (must be before StaticFiles)
6. **Static Files** — with immutable 1-year cache for `/assets/`, `/css/`, `/scripts/`, `/js/` and 7-day cache for `/uploads/`
7. **WebMarkupMin** — HTML minification + Brotli/GZip compression
8. **Output Cache** — stores rendered responses in memory (120s TTL)
9. **Cookie Policy**
10. **uTPro Middleware** — request localization (culture cookie)
11. **Umbraco Pipeline** — BackOffice + Website routing
12. **MapControllers** — MVC endpoint routing

> **Important:** `UseWebOptimizer()` must come before `UseStaticFiles()`. WebOptimizer intercepts CSS/JS requests to minify them. If StaticFiles runs first, it serves the original unminified file and WebOptimizer never sees the request.

# 3.9 Script Queue — Loading JS from Block Components

Block grid components often need their own JavaScript. uTPro provides a **Script Queue** system (`ScriptQueueExtensions`) that lets components register JS files, and the Layout renders them in the correct loading position.

## Why not just use `<script>` in the component?

The Layout uses a **deferred loader** that loads jQuery + plugins only after user interaction (or 3s idle). If a component renders its own `<script defer>` tag, it may execute **before** jQuery is available — causing `$ is not defined` errors.

## Two queues

| Method | When to use | How it loads |
|--------|-------------|--------------|
| `Html.QueueScript(path)` | Script **needs jQuery** or other deferred libraries | Appended to the sequential deferred loader array (loads after jQuery) |
| `Html.QueueStandaloneScript(path)` | Script has **no dependencies** (vanilla JS) | Rendered as a separate `<script defer>` tag |

Both methods **deduplicate automatically** — if the same component appears twice on a page, the script is only loaded once.

## Usage in a component `.cshtml`

```razor
@* Script that needs jQuery *@
@{ Html.QueueScript($"/scripts/{CurrentSite.GetItem().Root.Name}/slider.js"); }

@* Standalone script (no jQuery needed) *@
@{ Html.QueueStandaloneScript($"/scripts/{CurrentSite.GetItem().Root.Name}/cookie-consent-banner.js"); }
```

## What happens in the Layout

```html
<!-- Deferred loader (jQuery + plugins + component scripts via QueueScript) -->
<script>
    var scripts = [
        '/assets/scripts/jquery.min.js',
        ...
        '/scripts/uTPro/main.js',
        '/scripts/uTPro/slider.js'   ← added by component via QueueScript
    ];
    // loads sequentially after interaction/idle
</script>

<!-- Standalone scripts (via QueueStandaloneScript) -->
<script src="/scripts/uTPro/cookie-consent-banner.js" defer></script>
```

## Rules

1. **Vanilla JS** (no dependencies) → use `QueueStandaloneScript`
2. **Needs jQuery** → use `QueueScript`
3. **Never** put `<script src="...">` directly in a component — use the queue instead
4. CSS can still use `SetSection` as before (CSS doesn't have dependency ordering issues)
