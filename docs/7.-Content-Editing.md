# 7. Content Editing Guide

This guide is for **content editors** who build and maintain the website in the Umbraco backoffice. It covers the content tree, how pages are assembled from Block Grid components, the component catalogue, navigation, SEO and multilingual content.

> Developer helpers used inside views are documented separately in [9. Developer Reference](9.-Developer-Reference). Settings that live in dedicated backoffice nodes (favicon, robots, SMTP, Turnstile…) are in [8. Global Settings](8.-Global-Settings).

## 7.1 Signing in

Open `/umbraco` on your site (or the dedicated backoffice domain if one is configured — see [4.2](4.-Configurations#42-configuration-backoffice-and-website-run-separately)) and sign in with your backoffice credentials. Content lives in the **Content** section; uploaded files in **Media**; translations in **Dictionary** (Translation section).

## 7.2 Content tree

uTPro organises content around a shared root and per-site nodes. A typical tree looks like this:

```
Global Root                         (container, no URL)
├── Folder Sites                    (container, no URL)
│   └── Home                        ← the site root (Page Home)
│       ├── About / Services / …    (Page Standard pages)
│       ├── Folder Navigation Link  (holds the menu — see 7.5)
│       └── Page Error              (404 / 500 content — see 8.6)
└── Folder Settings                 (container, no URL)
    ├── Site Settings               (favicon, logo, title suffix)
    └── Sitemap & Robots Settings   (robots.txt, sitemap rules)
```

Container nodes (`Global Root`, `Folder Sites`, `Folder Settings`, `Folder Navigation Link`) are **organisational only** — they don't produce a public URL. Their "Links" panel shows a message instead of a URL (handled by `HiddenUrlProvider`; see [8.5](8.-Global-Settings#85-hidden-container-urls)).

The **Home** node is the site root and carries site-wide settings (SEO defaults, navigation link, culture & hostnames).

## 7.3 Page types

| Doc type | Alias | Use for |
|----------|-------|---------|
| **Page Home** | `uTPro__PageHome` | The single site root. Adds navigation config (menu link, 404 page) on top of the standard page fields. |
| **Page Standard** | `uTPro__PageStandard` | Every normal content page (About, Services, article listings…). |

Both page types share the same layout model (`uTPro__ContentLayout`) and render three Block Grid areas in order:

1. **Top Component** (`topComponent`) — inherited down the tree (set once on a parent, applies to descendants).
2. **Content** (`content`) — the page's own body, set per page.
3. **Bottom Component** (`bottomComponent`) — inherited down the tree.

> "Inherited" means a Top/Bottom block set on `Home` automatically appears on child pages unless a child sets its own. Use this for site-wide headers/footers/CTAs.

## 7.4 Building a page with the Block Grid

1. Open a page and go to the **Content** property (the Block Grid editor).
2. Click **Add content** and pick a component (see the catalogue in [7.6](#76-component-catalogue)).
3. Fill in the component's fields, then **Save** (draft) or **Save and publish** (live).
4. Reorder blocks by dragging; delete via the block's context menu.
5. Wrap blocks in a **One Column Section** to control the section background/colours (next section).

### Section wrapper — colours & background

`globalOneColumnSection` ("One Column Section") is a layout block that wraps content and drives its colours from the CMS (rendered via `wrapper.css`):

| Field | Alias | Purpose |
|-------|-------|---------|
| Title | `title` | Optional section heading |
| Background Color | `backgroundColor` | Section background (colour picker) |
| Background Image | `backgroundImage` | Optional background image |
| Background Opacity | `backgroundOpacity` | Overlay opacity for the background image |
| Text color | `textColor` | Section text colour |

> Text colour falls back to a safe theme value (`--utp-text`) if you leave it blank, so text stays readable. Brand colours themselves live in `variables.css` (developer-side); see [4.6](4.-Configurations#46-seo-configuration).

## 7.5 Navigation menu

The header menu is **not** hard-coded — editors build it:

1. Under **Home**, open the **Folder Navigation Link** node.
2. Its **Items** property (`navigationItems`) is a list of **Global Navigation Item** blocks.
3. Add one block per menu entry.

Each **Global Navigation Item** (`globalNavigationItem`) has:

| Field | Alias | Notes |
|-------|-------|-------|
| Link | `navLink` | Target page or external URL (label comes from the link name) |
| Inner CSS Class | `navInnerCssClass` | Class on the `<a>`. The special value **`nav-logo-href`** renders the logo image instead of a text label |
| Outer CSS Class | `navOuterCSSClass` | Class on the `<li>` |
| Hidden Item | `navHiddenItem` | Hide this entry without deleting it |
| Hide label | `navHideLabel` | Render the link with no text |
| Children | `navChildren` | Nested items for multi-level menus |
| Add "noopener" / "noreferrer" | `navAddNoopenerToLink` / `navAddNoreferrerToLink` | Security attributes for external links |
| Hide if member logged in / out | `navHideIfMemberIsLoggedIn` / `navHideIfMemberIsLoggedOut` | Member-aware visibility |

The active menu item is detected automatically (current page or an ancestor). When the site has **more than one language**, a **language switcher** is added to the menu automatically (see [7.8](#78-multilingual-content)).

## 7.6 Component catalogue

All components below are Block Grid element types (alias prefix `uTPro__`). Rich-text fields accept formatted HTML; "Link" fields use the Umbraco link picker.

### Banner Header — `uTPro__BannerHeader`
Hero banner at the top of a page.
- **Title** (`title`), **Description** (`description`)

### Introduction — `uTPro__Introduction`
Intro block with a call to action.
- **Highlight** (`highlight`), **Special** (`special`, rich text), **Description** (`description`, rich text), **Call to Action** (`callToAction`, link)

### Introduction Detail — `uTPro__IntroductionDetail`
Feature-rich intro with image and a feature list.
- **Title** (`title`), **Description** (`description`), **Image** (`image`)
- **List Feature** (`listFeature`) — a list of **Feature** blocks
- **Call to Action Get Started** (`callToActionGetStarted`), **Call to Action More Info** (`callToActionMoreInfo`)

**Feature** (`uTPro__IntroductionDetail__Feature`): **Title** (`title`), **Class Icon** (`classIcon`, a Font Awesome class), **List** (`list`, multiple text lines)

### Endorsements — `uTPro__Endorsements`
Testimonials / partner highlights.
- **Title** (`title`), **Highlight** (`highlight`) — a list of **Highlight** blocks

**Highlight** (`uTPro__Endorsements__Highlight`): **Title** (`title`), **Description** (`description`), **Image** (`image`), **Link** (`link`)

### Contact Form — `uTPro__ContactForm`
Contact section with a form and contact details.
- **Contact Form** (`contactForm`) — selects which Simple Form to render (see [8.4](8.-Global-Settings#84-contact-form--cloudflare-turnstile))
- **Address** (`address`, rich text), **Email** (`email`, links), **Phone** (`phone`, links), **Social** (`social`, links)

### Article — `uTPro__Article`
Long-form article block.
- **Title** (`title`), **Description** (`description`), **Thumbnail** (`thumbnail`)
- **Content** (`content`) — a list of **Image** and **Paragraph** blocks:
  - **Article Image** (`uTPro__Article__Image`): **Image** (`image`), **Alt** (`alt`)
  - **Article Paragraph** (`uTPro__Article__Paragraph`): **Richtext** (`richtext`)

### Cookie Consent Banner — `uTPro__CookieConsentBanner`
Cookie notice text.
- **Content** (`content`, rich text)

## 7.7 SEO per page

Each page has SEO fields (from the [UmbracoSeoVisualizer](https://marketplace.umbraco.com/package/umbracoseovisualizer) package):

- **Title and Description** (`headTitleAndDescription`) — search-result preview. The site-wide **Title Suffix** (Site Settings, `siteSettingsTitleSuffix`) is appended as `Page Title | Site Name`.
- **Keywords** (`headKeywords`), **Social Share Image** (`headSocialShareImage`, ≥1200×630).
- **Open Graph** (`ogTitle` / `ogDescription` / `ogImage` / `ogType`, `facebookAppId`) and **Twitter Card** (`twitterCardType` / `twitterTitle` / `twitterDescription` / `twitterImage` / `twitterImageAlt` / `twitterSite`).
- **Sitemap & indexing** (composition `GlobalPageSitemapSetting`):
  - **Hidden Sitemap** (`sitemapHiddenSitemap`) — adds `noindex` and removes the page from `sitemap.xml`.
  - **Hidden their children** (`sitemapHiddenTheirChildren`) — also excludes child pages.
  - **Stop internal links being followed** (`sitemapStopInternalLinksBeingFollowed`) — adds `nofollow`.
  - **Change frequency** (`sitemapXmlChangeFrequency`), **Priority** (`sitemapXmlPriority`).

Canonical URLs, hreflang and JSON-LD are generated automatically — see [4.6](4.-Configurations#46-seo-configuration).

### URLs & redirects (Page Home fields)
- **Alternative name / URL** (`umbracoUrlName` / `umbracoUrlAlias`) — override the URL segment or add extra URLs.
- **Content Redirect** (`umbracoInternalRedirectId`) — serve another node's content without changing the URL.
- **URL Redirect** (`umbracoRedirect`) — 302 redirect to another node.
- **Custom scripts** (`headerOpeningScripts` / `headerClosingScripts` / `bodyOpeningScripts` / `bodyClosingScripts`) — inject per-page `<head>`/`<body>` markup (e.g. analytics).

## 7.8 Multilingual content

uTPro supports multiple languages via Umbraco culture variants plus cookie-based language memory:

1. Configure languages under **Settings → Languages**.
2. Set **Culture and Hostnames** on the **Home** node (one hostname per language). See [4.1](4.-Configurations#41-configuration-remember-language---culture-and-hostnames) for the important ordering rules.
3. Edit each page's variant per language (the language dropdown at the top of the editor).
4. Use **Dictionary items** (Translation section) for shared UI strings; reference them in views with `CurrentSite.GetDictionaryValue("Key")` (see [9](9.-Developer-Reference)).

When more than one language is active, the header renders a **language switcher** automatically, and the current language is remembered via cookie.

## 7.9 Media & responsive images

- Upload files in the **Media** section, then pick them in image/link fields.
- Images rendered through uTPro's responsive helper are automatically resized and served as **WebP** with a `srcset`. You control this from **Responsive Image Settings** — see [8.3](8.-Global-Settings#83-responsive-image-settings).
- For best Core Web Vitals, use appropriately sized source images; the hero/LCP image is preloaded automatically.

## 7.10 Publishing checklist

- **Save** stores a draft; **Save and publish** makes it live.
- Set SEO **Title and Description** before publishing.
- Check the page in **Preview** (Block Preview renders components as they'll appear — see [10.3](10.-Backoffice-Tools#103-block-preview)).
- After changing site-wide settings (favicon, robots, navigation), the public site refreshes via cache; allow up to the output-cache TTL (120s) or clear the cache.
