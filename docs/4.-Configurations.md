# 4.3 uTPro Custom Configuration

All uTPro-specific settings are under the `uTPro` section in `appsettings.json`:

```json
{
  "uTPro": {
    "Backoffice": {
      "Enabled": true,
      "Url": "bo.yourdomain.com"
    },
    "RememberLanguage": {
      "Enabled": true,
      "ListExludeRequestLanguage": {
        "Enabled": true,
        "Paths": []
      }
    }
  }
}
```

| Key | Description |
|-----|-------------|
| `uTPro:Backoffice:Enabled` | Enable separate backoffice domain |
| `uTPro:Backoffice:Url` | Backoffice domain(s), comma-separated |
| `uTPro:RememberLanguage:Enabled` | Enable cookie-based language persistence |
| `uTPro:RememberLanguage:ListExludeRequestLanguage:Enabled` | Enable path exclusion for language detection |
| `uTPro:RememberLanguage:ListExludeRequestLanguage:Paths` | Array of paths to exclude from language detection |

# 4.4 Security Configuration

uTPro includes built-in security hardening in `Program.cs`:

- **Security Headers** (production): `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Permissions-Policy`, `Strict-Transport-Security`
- **Session Cookies**: `HttpOnly`, `Secure`, `SameSite=Strict`, 30-minute idle timeout
- **Request Size Limits**: 128MB max upload, 4MB per form value, 10240 max form values
- **Data Protection**: Keys persisted to `umbraco/Data/TEMP/PersistKeys/` with 90-day rotation
- **Domain Policy**: Configurable domain allowlist with wildcard support in `CheckPolicy.cs`
- **HSTS**: Enabled in production with 1-year max-age and includeSubDomains

# 4.5 Performance Configuration

- **WebOptimizer**: CSS/JS minification for `css/**/*.css`, `js/**/*.js`, `assets/**/*.css`, `assets/**/*.js`
- **WebMarkupMin**: HTML minification + Brotli/GZip/Deflate compression (10MB max response size)
- **Razor Runtime Compilation**: Enabled for development flexibility
- **Kestrel**: Server header disabled for security

# 4.6 SEO Configuration

SEO is managed through Umbraco CMS content nodes with the following structure:

- **Title & Description**: Per-page via SeoVisualizer, with site-level suffix template (`<titlePage> | <titleSiteName>`)
- **Open Graph**: `og:type`, `og:title`, `og:description`, `og:image` (auto absolute URL), `og:site_name`, `og:locale`
- **Twitter Cards**: `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image` (auto absolute URL)
- **Canonical URL**: Auto-generated absolute URL per page
- **Hreflang**: Auto-generated alternate language links for multilingual sites
- **Robots Meta**: Per-page noindex/nofollow control via `SitemapStopInternalLinksBeingFollowed`
- **JSON-LD**: WebSite structured data schema auto-generated
- **Sitemap**: Auto-generated at `/sitemap.xml`
- **Favicon**: Dynamic from CMS settings with fallback to `/favicon.ico`
- **Error Pages**: Include `noindex, nofollow` meta tag
