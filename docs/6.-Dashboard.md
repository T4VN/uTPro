# 6. Dashboard & Header App

`uTPro.Feature.Dashboard` adds a branded **backoffice dashboard** and a **header app** to Umbraco. Both are backed by a single authenticated management-API controller, so nothing is exposed publicly and the GitHub "latest release" lookup happens server-side (cached) — the browser never calls GitHub directly.

## 6.1 What You Get

| Extension | Where | Purpose |
|-----------|-------|---------|
| **Header App** (`utpro-header-app`) | Top-right of the backoffice header | Popover card with the uTPro logo, installed version, an **Update** or **Check for Update** action, and a shortcut to the dashboard |
| **Dashboard** (`utpro-dashboard`) | Content section → **uTPro** tab | Welcome/overview page: version + update status, site statistics, current-user details, and recent activity / audit trail |

Both are registered in `App_Plugins/utpro-header/umbraco-package.json`. The dashboard is shown in the Content section (`Umb.Condition.SectionAlias` = `Umb.Section.Content`).

## 6.2 Dashboard Layout

The dashboard uses a two-column layout (stacks on narrow screens):

- **Left (main)**
  - **Hero** — logo, product blurb, and either an "Update to vX" button or an "Up to date" tag + "Check for Update".
  - **Resources** — quick links (Documentation, Releases, Report an issue, Umbraco docs), built from the server-provided repo URLs.
  - **Recent trail** — two cards (see [6.4](#64-recent-activity--audit-trail)).
  - **Recent activity** — two cards.
- **Right (sidebar)**
  - **Product Information** — product name, core (Umbraco) version, installed version, latest release.
  - **Site** — published content, items in recycle bins, users/members totals and disabled counts.
  - **Your Details** — name, email, 2FA status, last login, last password change.

## 6.3 Version & Update Check

The header app and dashboard share version state via `config.js`:

- `GET version` returns `installed`, `latest`, `updateAvailable`, plus `website` / `releasesUrl`.
- The latest release tag is fetched from GitHub **server-side** and cached for 1 hour (avoids CORS, proxies, and rate limits).
- `GET version?refresh=true` (the "Check for Update" button) drops the server cache and re-checks immediately.
- Versions are normalized for display (`v17.0.4` → `17.0.4`).

## 6.4 Recent Activity & Audit Trail

The dashboard surfaces two kinds of history, each in a **"Your …"** (current user) and **"All …"** (everyone) card.

### Recent activity — from `umbracoLog`

Content-style actions (save, publish, create, move, delete, …). Each entry resolves the affected node's name and links to its editor:

| Entity type | Link target |
|-------------|-------------|
| `Document` | `/umbraco/section/content/workspace/document/edit/{key}` |
| `Media` | `/umbraco/section/media/workspace/media/edit/{key}` |
| `DictionaryItem` | `/umbraco/section/translation/workspace/dictionary/edit/{key}` |

> **Dictionary items** are not `umbracoNode` rows — their `NodeId` maps to `cmsDictionary.pk`. The controller joins `cmsDictionary` and resolves the name (the dictionary key, e.g. `uTPro.Form.ContactUs.Message`) and edit guid from there.

Each row shows the action, a status badge (Saved / Created / Published / …), and a meta line with the timestamp, the user (in the "All" card), and a friendly **entity type** label (`Document` → *Content*, `DictionaryItem` → *Dictionary*, …).

### Recent trail — from `umbracoAudit`

Cross-cutting events (sign-in, save, user management, …). Each row shows the event details, a meta line (timestamp + user), and the raw `eventType` as a tag.

> Audit-trail timestamps are stored in **UTC**; activity (`umbracoLog`) timestamps are stored in **local server time**. They are formatted with the browser locale, so the two rows can differ by your server's UTC offset.

### Collapsible cards

Every activity/trail card can be **collapsed or expanded independently** via the caret (`uui-symbol-expand`) in its header. By default the **trail cards start collapsed** and the **activity cards start expanded**. Collapsing a card shrinks it to just its header without affecting its neighbour.

## 6.5 Management API

All endpoints live under `/umbraco/management/api/v1/utpro/dashboard/` and require backoffice authentication.

| Endpoint | Returns |
|----------|---------|
| `GET version` (`?refresh=true`) | Installed + latest version and update status |
| `GET stats` | Site statistics (content, recycle bins, users, members) |
| `GET current-user` | Current backoffice user details |
| `GET recent-activity` / `GET my-activity` | Newest `umbracoLog` entries (all users / current user) |
| `GET recent-trail` / `GET my-trail` | Newest `umbracoAudit` entries (all users / current user) |

Activity/trail endpoints return the newest 20 rows (`ActivityTake`).

## 6.6 File Locations

| File | Role |
|------|------|
| `Controllers/uTProDashboardManagementController.cs` | All API endpoints; SQL for stats, activity (`umbracoLog`) and trail (`umbracoAudit`) |
| `App_Plugins/utpro-header/umbraco-package.json` | Registers the header app + dashboard extensions |
| `App_Plugins/utpro-header/config.js` | Shared config + authenticated `fetch*` helpers, version caching |
| `App_Plugins/utpro-header/dashboard.element.js` | Dashboard element (layout, cards, collapse logic, node links) |
| `App_Plugins/utpro-header/header-app.element.js` | Header popover element |

> The `App_Plugins` files are static assets — after editing them, hard-refresh the backoffice to bypass the browser cache.
